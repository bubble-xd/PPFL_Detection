# 本次改动说明

## 1. 改动背景

当前 `selected_layers` 的特征构造方式是：

1. 按 `KEY_LAYER_MAP` 选出若干层前缀。
2. 直接把这些层对应的 `delta` 参数展平后拼接。
3. 把拼接后的大向量直接送给 Krum 等聚合器做距离计算。

这样做有一个明显问题：

- 大层会天然占据更多维度，进而主导欧氏距离。
- 对 `resnet34` 当前的关键层配置 `['fc', 'layer4.2.conv2', 'conv1', 'layer4.2.conv1']` 来说，
  `layer4.2.conv1` 和 `layer4.2.conv2` 两个大卷积层几乎会吞掉全部几何结构，
  `fc` 和 `conv1` 的作用会被严重削弱。

因此，这次改动的目标是：

- 在线上特征构造阶段，先消除“层维度大小”带来的不公平权重。
- 保持现有离线筛层结果不变，先验证问题是不是主要出在“拼接几何”。

另外，这里也补充记录一个与 offline `layer_extraction` 相关的既有修正：

- 离线关键层提取里，`magnitude` 和 `cosine distance` 都会先做跨层 `Z-score`
- 如果再直接用 `Z-score` 之后的方差做动态权重分配，那么这两个方差在大多数轮次都会非常接近 `1`
- 结果就是 `alpha / beta` 很容易退化到接近 `0.5 / 0.5`
- 这样“自适应赋权”会在数值上基本失效

因此，offline 打分阶段后来改成了：

- `Z-score` 仍然负责做无量纲对齐
- 但动态权重 `alpha / beta` 改为根据**原始 magnitude / cosine distance 的跨层方差**来计算

也就是：

`alpha_t = Var(m) / (Var(m) + Var(c) + eps)`

`beta_t = 1 - alpha_t`

最后再去组合：

`s_l^(t) = alpha_t * hat(m)_l^(t) + beta_t * hat(c)_l^(t)`

这部分对应的实现和说明分别在：

- [layer_extraction/scoring.py](/home/user/PPFL_detection/layer_extraction/scoring.py)
- [layer_extraction/README.md](/home/user/PPFL_detection/layer_extraction/README.md)

当前导出的 `selection.json` 里也会记录：

- `weighting_mode = "raw_variance_on_zscored_scores"`

## 2. 新增特征模式

本次新增了两个特征模式：

- `selected_layers_balanced`
- `selected_layers_balanced_projected`

### 2.1 `selected_layers_balanced`

对每个选中层前缀 `l`，先取该层对应的 `delta_l`，然后做：

`delta_l <- delta_l / sqrt(dim_l)`

其中：

- `dim_l` 是该层块的参数总维度
- 缩放按“层块”统一进行，而不是按单个参数张量单独进行

最后再按原有顺序拼接成总特征向量。

这个设计的含义是：

- 每一层先在自己的维度尺度上做对齐
- 大层不再仅凭“参数更多”就主导客户端间距离
- 离线筛层时强调的“层级差异”能更自然地保留到线上检测阶段

### 2.2 `selected_layers_balanced_projected`

在 `selected_layers_balanced` 的基础上，再做一次随机投影：

`balanced_selected_vector -> projection`

也就是说，现在的顺序变成：

1. 先按层平衡
2. 再拼接
3. 再投影

而不是旧的：

1. 先原始拼接
2. 再投影

这样能避免“先被大层几何带偏，再投影”的问题。

这是本次改动的核心文件。

- 新增 `_build_key_scale_map_for_prefixes()`

### 3.4 offline `layer_extraction` 的动态权重修正

这部分不是本轮线上 `balanced` 几何修复直接新增的代码，但这里一并补充记录，避免后面读文档时遗漏。

offline 关键层提取的原始思路是：

1. 先分别计算每层的 `magnitude` 和 `cosine distance`
2. 对两者都做跨层 `Z-score`
3. 再根据两类特征的“方差”分配动态权重

问题在于：

- 如果权重也是基于 `Z-score` 之后的方差来算，
  那么这两个方差在大多数情况下都会非常接近 `1`
- 于是动态权重 `alpha / beta` 会塌缩到接近 `0.5 / 0.5`
- 最终“动态赋权”名义上存在，但实际几乎失去作用

所以这里后来改成：

- `Z-score` 只负责做无量纲对齐
- 权重分配改为基于**原始特征值的跨层方差**

也就是：

- `alpha_t = Var(magnitudes) / (Var(magnitudes) + Var(cosine_distances) + eps)`
- `beta_t = 1 - alpha_t`

这样做的原因是：

- `Z-score` 后的值适合组合
- 原始方差更适合反映“这一轮哪类特征更有区分度”

当前实现里，这个逻辑对应 `compute_adaptive_weights()`，
最终导出的 `selection.json` 中会把模式标记为：

- `raw_variance_on_zscored_scores`

## 6. 建议的效果验证顺序

为了判断问题到底是“拼接几何”还是“筛层准则”，建议按下面顺序看结果：

1. 固定当前 `KEY_LAYER_MAP`
2. 先比较以下模式：
   - `raw_full`
   - `selected_layers`
   - `selected_layers_balanced`
   - `selected_layers_projected`
   - `selected_layers_balanced_projected`
3. 重点看：
   - `F1`
   - `BM-Gap`
   - final `ACC`
   - final `ASR`
4. 如果 `balanced` 相比 `selected_layers` 明显变好，
   说明主要问题来自原始拼接几何
5. 只有在几何修正后，再去扫 `K=4/6/8/10`
